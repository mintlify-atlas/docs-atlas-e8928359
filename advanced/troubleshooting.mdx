---
title: "Troubleshooting"
description: "Solutions to common issues with TTS Voice Wizard."
---

<AccordionGroup>
  <Accordion title='"Configuration Failed to Initialize" on startup'>
    Your settings file has become corrupted. TTS Voice Wizard automatically creates a backup on each launch.

    **To fix:**
    1. When prompted, click **Yes** to open `AppData\Local\TTSVoiceWizard`
    2. Delete the `user.config` file (and any `.bak` files if the backup is also corrupt)
    3. Restart TTS Voice Wizard — settings will reset to defaults

    If this happens repeatedly, try running the app as Administrator or moving it to a directory where your user account has full write permissions.
  </Accordion>

  <Accordion title="VRChat chatbox not showing text">
    1. In VRChat, open **Action Menu → Options → OSC → Enable** and confirm OSC is enabled
    2. In TTS Voice Wizard, verify the OSC address is `127.0.0.1` and port is `9000`
    3. Check Windows Firewall is not blocking UDP on port 9000
    4. Ensure the **Send VRChat Chatbox** toggle is enabled in TTS Voice Wizard
    5. Try restarting VRChat — OSC sometimes needs a fresh session to activate
  </Accordion>

  <Accordion title="No audio from TTS">
    1. Confirm your audio output device is correctly selected in TTS Voice Wizard settings
    2. Check the volume slider in the app is not at zero
    3. Ensure the **Disable TTS** toggle is off
    4. Try switching to **System Speech** as a baseline test — it requires no API key
    5. If using a virtual cable, verify the cable is installed and selected correctly (see [Virtual Cable](/advanced/virtual-cable))
  </Accordion>

  <Accordion title="Speech recognition not working">
    1. Check that Windows has granted microphone permissions to TTS Voice Wizard (**Windows Settings → Privacy → Microphone**)
    2. Confirm the correct microphone is selected in the app
    3. For **Azure STT**: verify your API key and region (default region: `eastus`)
    4. For **Windows Built-in**: ensure Windows Speech Recognition is set up (**Control Panel → Ease of Access → Speech Recognition**)
    5. For **Vosk / Whisper**: confirm the model files have been downloaded and placed in the correct folder
  </Accordion>

  <Accordion title="Windows SmartScreen blocks the app">
    Windows may warn about unsigned executables from the internet.

    1. Click **More info** on the SmartScreen dialog
    2. Click **Run anyway**

    Alternatively, right-click the `.exe` → **Properties** → check **Unblock** → click **OK**.
  </Accordion>

  <Accordion title="OSC not receiving heart rate or battery data">
    1. Confirm the OSC Listener receive port in TTS Voice Wizard is `4026`
    2. Verify XSOverlay or HRtoVRChat_OSC is configured to send OSC to port `4026`
    3. Check Windows Firewall allows UDP inbound on port `4026`
    4. Ensure the OSC Listener toggle is enabled in TTS Voice Wizard
  </Accordion>

  <Accordion title="Spotify not connecting">
    1. Make sure Spotify is installed and you are logged in
    2. In TTS Voice Wizard, click **Disconnect** then **Connect Spotify** again to restart the OAuth flow
    3. If the browser doesn't open automatically, check that your default browser is set
    4. Verify your internet connection — the PKCE token exchange requires network access
  </Accordion>

  <Accordion title="High CPU usage">
    **Whisper** and **Vosk** run AI inference locally on your CPU, which is CPU-intensive by design.

    - Switch to a lighter Whisper model (tiny or base) for lower CPU use
    - Use a cloud STT engine (Azure, Deepgram) to offload processing
    - Ensure you have closed other CPU-heavy applications
  </Accordion>

  <Accordion title="Settings not saving between sessions">
    1. Try running TTS Voice Wizard as Administrator
    2. Check that your user account has write access to `AppData\Local\TTSVoiceWizard`
    3. If settings reset on every launch, the config file may be on a read-only path — move the app folder to a writable location
  </Accordion>
</AccordionGroup>
